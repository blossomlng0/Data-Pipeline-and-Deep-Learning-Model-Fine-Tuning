# Data-Pipeline-and-Deep-Learning-Model-Fine-Tuning
# ✈️ 항공사 고객 만족도 예측 및 서비스 개선 프로젝트

> **KT AIVLE School 1차 미니프로젝트**
> * **진행 기간**: 2026.04.14 - 2026.04.15 (2일)
> * **주요 역할**: 딥러닝 모델링 및 파인튜닝, 데이터 전처리 파이프라인 함수화, 발표 PPT 제작
> * **사용 기술**: Python, TensorFlow/Keras, Pandas, NumPy, Scikit-learn, PowerPoint

<br>

## 📌 1. 프로젝트 개요 (Project Overview)
항공사 서비스 이용 고객의 설문 데이터를 기반으로 **고객 만족도(만족/불만족)를 예측하는 딥러닝 모델을 구축**하고, 변수 중요도 분석을 통해 **고객 만족도에 영향을 미치는 핵심 요인을 도출**하여 데이터 기반의 서비스 개선 전략을 제안한 프로젝트입니다.

<br>

## 🛠️ 2. 핵심 수행 내용 및 결과 (Key Deliverables & Results)
* **전처리 파이프라인 모듈화**: 결측치 처리, Target 인코딩, 라벨 인코딩, 가변수화, 스케일링을 일괄 처리하는 재사용 가능한 함수 설계
* **딥러닝 모델 학습 및 최적화**: 추가 수집된 데이터를 통한 파인튜닝(Fine-tuning) 및 Early Stopping 적용으로 안정적인 학습 곡선 확보
* **최종 성과**: 불만족 고객 예측 성능을 크게 보완하여 **최종 예측 정확도(Accuracy) 93%** 달성
* **인사이트 도출**: 세대별 타겟 전략을 위해 연령대별 변수 중요도(Feature Importance) 분석 수행

<br>

## 🚨 3. 문제 상황 및 해결 과정 (Troubleshooting)

### ❓ Problem: 클래스 불균형 및 데이터 구조 불일치
1. **예측력 저하**: 기존 모델이 '만족' 데이터에 치우쳐 있어 비즈니스 개선에 핵심이 되는 **'불만족(0)' 고객에 대한 예측 성능이 상대적으로 낮은 문제**가 있었습니다.
2. **전처리 오버헤드**: 추가 설문 데이터를 반영하는 과정에서 학습(Train) 데이터와 검증(Validation) 데이터 간의 범주형 변수 카테고리 구조가 달라 **전처리 오류가 반복적으로 발생**했습니다.

### 💡 Solution: 데이터 재학습 및 전처리 파이프라인 구축
1. **모델 성능 최적화**: 추가 수집된 균형 잡힌 데이터를 기반으로 모델 재학습 및 파인튜닝을 진행하여 과적합을 방지하고 불만족 고객에 대한 정밀한 예측력을 확보했습니다.
2. **파이프라인 함수화**: 범주형 데이터 전처리 시 **카테고리를 강제로 고정하는 로직을 포함한 일괄 처리 함수**를 구현했습니다. 이를 통해 신규 데이터 유입 시 발생할 수 있는 구조적 오류를 완벽히 해결하고 데이터 처리 일관성을 확보했습니다.

<br>

## ✨ 4. 차별점 및 배운 점 (Our Strengths & Lessons Learned)

### 🏃‍♂️ 우리 조만의 차별성
* **엔지니어링 관점의 접근**: 타 조가 하이퍼파라미터 튜닝과 단순 성능 올리기에 집중할 때, 실제 프로덕션 환경을 고려하여 **재사용 및 확장 가능한 전처리 파이프라인 구조를 설계**했습니다.
* **비즈니스 관점의 세분화**: 만족도 요인을 통틀어 분석하지 않고 **연령대별 변수 중요도**를 세부 분석했습니다.
  * *35세 이하*: **'온라인 탑승 수속'** 만족도가 핵심 요인
  * *35세 이상*: **'기내 와이파이 서비스'** 만족도가 핵심 요인
  * 세대별로 다르게 작용하는 가치를 도출하여 실질적이고 차별화된 고객 관리 전략(PPT)을 시각화했습니다.

### 🧠 배운 점
* 데이터 전처리 방식과 파이프라인 구조 설계가 모델의 최종 성능과 업무 효율성에 얼마나 지대한 영향을 미치는지 깊이 깨달았습니다.
* 단순히 모델의 Score를 높이는 것을 넘어, 변수 중요도 분석을 통해 데이터 뒤에 숨겨진 고객의 행동 패턴을 읽고 비즈니스 전략으로 연결하는 프로세스를 체득했습니다.
